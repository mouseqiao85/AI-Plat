## 一、项目概述

### 1.1 项目背景

当前智能体应用市场仅支持基础的智能体展示和简单创建功能，无法满足用户自定义复杂工作流的需求。为提升平台能力，需要将应用市场改造为低代码创建平台，支持可视化工作流编排、智能体配置、知识库管理等能力。

### 1.2 改造目标

- **可视化工作流编排**：拖拽式节点编排，降低使用门槛
- **智能体深度配置**：支持提示词、模型、工具、知识库等多维度配置
- **低代码开发**：从表单填写升级为可视化构建
- **快速迭代**：支持测试、调试、版本管理

### 1.3 核心价值

| 用户角色 | 痛点 | 解决方案 | 价值 |
|---------|------|---------|------|
| 业务人员 | 不会编程，无法创建智能体 | 可视化拖拽编排 | 零代码创建应用 |
| 开发者 | 需要写大量配置代码 | 模板+配置面板 | 提升开发效率10x |
| 企业用户 | 缺少定制化能力 | 自定义工作流 | 满足业务场景需求 |

#### 核心缺失 ❌

| 缺失能力 | 当前状态 | 影响 | 优先级 |
|---------|---------|------|--------|
| 可视化工作流编辑器 | ❌ 无 | 无法拖拽编排 | P0 |
| 智能体配置界面 | ⚠️ 仅表单 | 无法深度配置 | P0 |
| 工作流持久化 | ❌ 无 | 无法保存自定义流程 | P0 |
| 节点库与模板 | ❌ 无 | 用户不知道如何构建 | P1 |
| 测试调试面板 | ❌ 无 | 创建后无法测试 | P1 |
| 工作流运行时API | ❌ 无 | 前端无法调用LangGraph | P0 |
| 知识库管理 | ❌ 无 | 无法上传文档配置RAG | P1 |

### 2.2 核心洞察

> **后端已经具备强大的 LangGraph 引擎，但前端无法直接使用！**

**关键问题**：
- LangGraph 的节点和边是硬编码在代码中
- 前端无法动态配置工作流
- 智能体创建只有基础表单，无法利用 LangGraph 能力

**改造核心**：
```
前端可视化编辑器 
  → JSON 工作流定义 
  → 后端 LangGraph 动态构建 
  → 执行引擎
```

```python
# 工作流解析器
class WorkflowParser:
    def parse(workflow_json: dict) -> StateGraph
    def _build_node(node_def: dict) -> Callable
    def _build_edge(edge_def: dict)
    def _build_condition(condition: str) -> Callable

# 节点注册表
class NodeRegistry:
    def register(node_type: str, builder: NodeBuilder)
    def get(node_type: str) -> NodeBuilder
    def list_all() -> List[NodeType]

# 节点构建器
class NodeBuilder(ABC):
    @abstractmethod
    def build(config: dict) -> Callable

# 工作流执行引擎
class WorkflowExecutor:
    async def execute(workflow_id: str, inputs: dict) -> dict
    async def stream(workflow_id: str, inputs: dict) -> AsyncGenerator
```

### 4.2 新增表结构

```sql
-- 工作流版本表
CREATE TABLE workflows (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  agent_id INTEGER NOT NULL,
  version INTEGER NOT NULL,
  workflow_json TEXT NOT NULL,
  changelog TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  created_by INTEGER,
  FOREIGN KEY (agent_id) REFERENCES marketplace_agents(id),
  FOREIGN KEY (created_by) REFERENCES users(id),
  UNIQUE(agent_id, version)
);

-- 知识库表
CREATE TABLE knowledge_bases (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  name TEXT NOT NULL,
  description TEXT,
  file_urls TEXT DEFAULT '[]', -- JSON array
  vector_store_config TEXT DEFAULT '{}',
  index_status TEXT DEFAULT 'pending', -- pending, indexing, ready, failed
  document_count INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 工作流执行记录表
CREATE TABLE workflow_executions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  workflow_id INTEGER NOT NULL,
  session_id TEXT NOT NULL,
  user_id INTEGER,
  input_messages TEXT,
  output_messages TEXT,
  execution_time_ms INTEGER,
  token_usage TEXT, -- JSON
  status TEXT DEFAULT 'running', -- running, completed, failed
  error_message TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (workflow_id) REFERENCES workflows(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 节点模板表
CREATE TABLE node_templates (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  category TEXT,
  description TEXT,
  node_type TEXT NOT NULL,
  default_config TEXT DEFAULT '{}',
  icon TEXT,
  is_builtin BOOLEAN DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

```python
# agent/app/workflow/parser.py
from typing import Dict, Any, Callable
from langgraph.graph import StateGraph, END
from app.graph.state import AgentState
from app.workflow.registry import NodeRegistry

class WorkflowParser:
    """解析前端JSON工作流定义，动态构建LangGraph"""
    
    def __init__(self):
        self.registry = NodeRegistry()
    
    def parse(self, workflow_json: Dict[str, Any]) -> StateGraph:
        """将JSON转换为LangGraph StateGraph"""
        
        # 验证工作流定义
        self._validate(workflow_json)
        
        # 创建StateGraph
        graph = StateGraph(AgentState)
        
        # 1. 添加所有节点
        for node_def in workflow_json['nodes']:
            node_builder = self.registry.get(node_def['type'])
            node_func = node_builder.build(node_def['config'])
            graph.add_node(node_def['id'], node_func)
        
        # 2. 添加所有边
        for edge_def in workflow_json['edges']:
            if 'condition' in edge_def:
                # 条件边
                condition_func = self._build_condition(edge_def['condition'])
                graph.add_conditional_edges(
                    edge_def['from'],
                    condition_func,
                    {edge_def['to']: edge_def['to']}
                )
            else:
                # 普通边
                graph.add_edge(edge_def['from'], edge_def['to'])
        
        # 3. 设置入口点
        start_node = self._find_start_node(workflow_json['nodes'])
        graph.set_entry_point(start_node['id'])
        
        return graph.compile()
    
    def _validate(self, workflow_json: Dict[str, Any]):
        """验证工作流定义的有效性"""
        # 检查必需字段
        if 'nodes' not in workflow_json or 'edges' not in workflow_json:
            raise ValueError("工作流必须包含 nodes 和 edges")
        
        # 检查是否有入口节点
        start_nodes = [n for n in workflow_json['nodes'] if n['type'] == 'trigger']
        if len(start_nodes) == 0:
            raise ValueError("工作流必须有一个 trigger 类型的入口节点")
        if len(start_nodes) > 1:
            raise ValueError("工作流只能有一个 trigger 类型的入口节点")
        
        # 检查节点ID唯一性
        node_ids = [n['id'] for n in workflow_json['nodes']]
        if len(node_ids) != len(set(node_ids)):
            raise ValueError("节点ID必须唯一")
        
        # 检查边的起点和终点是否存在
        node_id_set = set(node_ids)
        for edge in workflow_json['edges']:
            if edge['from'] not in node_id_set:
                raise ValueError(f"边的起点 {edge['from']} 不存在")
            if edge['to'] not in node_id_set:
                raise ValueError(f"边的终点 {edge['to']} 不存在")
    
    def _build_condition(self, condition: str) -> Callable:
        """构建条件函数"""
        def condition_func(state: AgentState) -> str:
            # 根据condition表达式求值
            # 例如："state['intent'] == 'query'"
            try:
                result = eval(condition, {"state": state, "True": True, "False": False})
                return "pass" if result else "fail"
            except Exception as e:
                # 条件求值失败，默认通过
                return "pass"
        
        return condition_func
    
    def _find_start_node(self, nodes: list) -> dict:
        """找到入口节点"""
        for node in nodes:
            if node['type'] == 'trigger':
                return node
        raise ValueError("未找到入口节点")
```

from typing import Dict, Any, Callable
from abc import ABC, abstractmethod

class NodeBuilder(ABC):
    """节点构建器基类"""
    
    @abstractmethod
    def build(self, config: Dict[str, Any]) -> Callable:
        """构建节点函数"""
        pass

class FunctionNodeBuilder(NodeBuilder):
    """函数节点构建器（包装现有节点）"""
    
    def __init__(self, node_func: Callable):
        self.node_func = node_func
    
    def build(self, config: Dict[str, Any]) -> Callable:
        # 可以根据config动态调整节点行为
        # 例如：注入提示词、模型配置等
        async def wrapped_node(state: Dict[str, Any]) -> Dict[str, Any]:
            # 注入配置到state
            state['node_config'] = config
            # 调用原始节点函数
            result = await self.node_func(state)
            return result
        
        return wrapped_node
