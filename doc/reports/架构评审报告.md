## 执行摘要

项目是一个 **小体量双语言原型**（Go 端 ~30 个 `.go`、Python 端 LangGraph 应用），核心思路清晰：Go 做 HTTP/SSE/JWT 网关，Python 做 Agent 执行层（FastAPI + LangGraph StateGraph）。但仓库当前状态**极度混乱**：被 2026-05-11 一次 AI 批量自优化的产物（20 个 Python 脚本 + 35 份 `agent_outputs/*.md` + 14 份根目录 `*.md`）严重污染，并存在 **4 枚明文真实密钥**、**重复的 `main()` 函数无法编译**、**README 声明与实际依赖脱节**三个阻塞性问题。

## 二、架构分析

### 2.1 总体拓扑

```
Client → Go Gateway(:8080, Gin/JWT/SSE)
           │
           ├── SQLite (data/agent.db)
           └── HTTP → Python Agent(:8001, FastAPI/LangGraph)
                       │
                       └── LLM (百度 clawguard / DeepSeek)

侧挂：hermes-bridge/ (FastAPI, 对接 Claude Code CLI)
```

### 2.2 Go 端骨架（`cmd/` + `internal/` + `pkg/`）
- `cmd/gateway/` — 入口（**问题**：包含 `main.go` 与 `main_secure.go` 两份 `package main`+`func main()`，同包重复符号会编译失败 — 见 §三.C1）
- `internal/api/` — 9 个 handler：auth / chat / admin / hermes / marketplace / metrics / skill / health（+ benchmark test）
- `internal/service/` — `agent / session / tool / user.go`；其中 **`session/` 是空目录**（未实现）
- `internal/middleware/` 与 `internal/api/middleware/` — **两套 CORS 中间件并存**，功能重叠
- `internal/store/` — sqlite / user / conversation / message / marketplace
- `pkg/` — `auth`（jwt）、`config`、`logger`
- `proto/agent.proto` — 仅 85 行定义、自标"future"、无 generated `.pb.go`

### 2.3 Python 端骨架（`agent/`）
`app/` 下子模块：`graph / harness / llm / memory / rag / skill / tools / workers / stream / core / api`。从命名看是 fin-assistant 同族项目的 Harness 五子系统复用（session / instructions / state / scope / validator）。依赖栈：FastAPI + LangGraph ≥0.2 + langchain-openai + SQLAlchemy async + aiosqlite + redis + passlib（`agent/pyproject.toml:4-21`）。

### 2.4 前端/其他
- `web/` — Vite + React 19 + antd 6（`web/package.json`）
- `ue/` — **仅一张 PNG 设计稿 + 根目录 `ue.txt`**，不是独立子系统
- `hermes-bridge/` — 独立 Python 服务，自带 `requirements.txt`（仅 fastapi/httpx/pydantic）

### 2.5 部署
- `deployments/docker/Dockerfile.agent` + `Dockerfile.gateway` ✅
- `deployments/k8s/{agent-service,gateway}.yaml` ✅
- `docker-compose.yml`（1 KB）
- **无 CI**：`.github/workflows/`、`.gitlab-ci.yml`、`Jenkinsfile` 均不存在
- **未纳入 git**：父仓库 `git status` 显示 `agent-platform/` 整目录为 `??` 未追踪

## 五、权衡判断

- **"AI 批量优化成果" vs 真实工程质量**：90 分钟写 14 份 md + 生成 35 份 agent_outputs，主题重复度极高，属于**伪装成进度的噪音**；清理优先级高于"保留历史"
- **Go + Python 双栈 vs 纯 Python**：当前 Go 端仅做路由/JWT/SQLite（约 30 个 .go 文件），与 Python Agent 重叠度可控，保留双栈合理；但 README 宣称的 Redis/PG/gRPC 必须要么补上要么删除
- **硬编码内网 IP + 明文密钥**：这是**原型早期把 .env 和源码同步 commit 的典型错误**，必须先轮换再讨论其他
- **`hermes-bridge` 作为独立进程** vs 合入 `agent/`：当前独立合理（对接外部 Claude Code CLI 进程），但三份 Python 依赖栈需统一（建议用 `uv workspace` 或 monorepo）
- **`ue/` + `ue.txt`**：不是功能模块，是需求稿；应移入 `doc/ux/`
